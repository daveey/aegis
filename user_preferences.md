# User Preferences

## Coding Style
- Prefer explicit over implicit.
- Use type hints everywhere.

## Communication
- Be concise.

## Asana Asset Rendering
- Always render Asana assets using the renderer format: `Name [type:gid]`.
- Use `t` for task, `p` for project, `u` for user, `s` for section, `pf` for portfolio, `tg` for tag.
